How Bakkboard works

Bakkboard never touches the game process. Easy Anti-Cheat never sees it — because there is nothing to see.

📊Stats & match data

Read from Rocket League's local Stats API — the same data the game exposes to the operating system. No process injection, no memory reading, nothing hooking into the game.

🎨Cosmetics swap

UPK files are patched on disk before the game launches. The game loads them through its normal file loading — exactly like loading any other game asset. Easy Anti-Cheat does not scan file content at rest.

🖥️Overlays

Separate windows rendered by the OS as normal applications. They sit on top of the screen just like a browser or Discord — completely outside the game process.

🗺️Workshop maps

Map files are copied to the correct game directory before launch, the same way the game expects them to be loaded. No runtime modification of any kind.

No ban risk. Bakkboard does not bypass or interfere with Easy Anti-Cheat in any way. It simply does not interact with anything the anti-cheat monitors. This is the fundamental difference from BakkesMod, which required DLL injection into the game process — the exact technique Easy Anti-Cheat blocks.
